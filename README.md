# Astra File Monitor (AFM)

Централизованный полнотекстовый поиск по пользовательским файлам на управляемых машинах и автоматическое выявление файлов, содержащих заданные ключевые слова (триггеры конфиденциальности), с немедленным оповещением администратора.

## Ключевые возможности

- 🔍 **Полнотекстовый поиск** с русской морфологией и гибкими фильтрами (владелец, расширение, размер, дата)
- 🛡️ **Автоматическое детектирование** конфиденциальных данных по настраиваемым триггерам
- 📊 **Интерактивные дашборды** с графиками инцидентов, типов файлов, доступности агентов
- 🤖 **Агенты** с реактивным мониторингом (inotify), REST API, плагинной системой и автообновлением
- 🖥️ **Веб-интерфейс** с современным тёмным дизайном, управлением агентами, массовыми действиями
- 📦 **Простое развёртывание** одной командой на любой Debian/Ubuntu/Astra Linux машине

## Требования

**Сервер:**
- Debian 11/12, Ubuntu 20.04/22.04/24.04 или Astra Linux SE 1.7
- root-доступ
- Порт 80 (HTTP) должен быть открыт для веб-интерфейса
- Минимальные ресурсы: 2 vCPU, 4 ГБ RAM, 20 ГБ дискового пространства

**Агенты:**
- Debian/Ubuntu/Astra Linux ARM (для рабочих станций)
- Python 3, доступ в интернет (или до сервера AFM)

## Быстрый старт (сервер)

1. **Склонируйте репозиторий и запустите установку**
   ```bash
   git clone https://github.com/Fedoroff00/afm-server.git
   cd afm-server
   sudo bash deploy.sh
   ```
   Скрипт автоматически:
   - установит Docker и Docker Compose (если они ещё не установлены);
   - определит ваш IP-адрес и сгенерирует надёжные пароли;
   - запустит все контейнеры (PostgreSQL, Redis, веб‑сервер, Celery);
   - предложит создать учётную запись администратора.

2. **Откройте веб-интерфейс**  
   Перейдите в браузере по адресу `http://<ваш‑IP>` и авторизуйтесь с логином/паролем, созданным при установке.

## Установка агентов на контролируемые машины

1. В веб‑интерфейсе перейдите в раздел **«Машины» → «Добавить агента»**.
2. Укажите хостнейм (и IP‑адрес) целевой машины, нажмите **«Создать агента»**.
3. На появившейся странице скопируйте готовую команду установки и выполните её **на целевой машине** (Debian/Ubuntu/Astra Linux ARM):

   ```bash
   wget http://<IP-сервера>/media/packages/astra-monitor-agent_latest_all.deb && \
   echo 'astra-monitor astra-monitor/server_url string http://<IP-сервера>' | sudo debconf-set-selections && \
   echo 'astra-monitor astra-monitor/token string ВАШ_ТОКЕН' | sudo debconf-set-selections && \
   sudo dpkg -i astra-monitor-agent_latest_all.deb
   ```

   Агент автоматически установится, запустится и начнёт сканировать `/home`, отправляя результаты на сервер.  
   Для массовой установки можно использовать системы управления конфигурациями (Ansible и т.п.).

### Как работает агент

- **Мониторинг в реальном времени** – отслеживает изменения в файловой системе через `inotify`, реагирует на создание/изменение файлов.
- **Периодическое полное сканирование** – раз в час (по умолчанию) проводит полный обход заданных директорий.
- **Извлечение текста** – поддерживаются текстовые форматы (`.txt`, `.log`, `.md` и др.) и конвертируемые через `pandoc` (`.odt`, `.docx`, `.rtf`, `.pdf`).
- **Отправка на сервер** – метаданные файла и его содержимое отправляются на центральный сервер для индексации и проверки триггеров.
- **Конфигурация через heartbeat** – агент получает актуальные настройки (список сканируемых директорий, исключения, лимиты) с сервера при каждом heartbeat.
- **Локальная очередь и повторные попытки** – при недоступности сервера данные сохраняются и отправляются после восстановления связи.
- **Встроенный REST API** – на порту `9090` можно получить метрики, статус, запустить сканирование или скачать файл напрямую.
- **Плагинная система** – можно добавлять свои обработчики инцидентов и событий в `/opt/astra-monitor/plugins`.

## Веб-интерфейс

После входа в систему доступны следующие разделы:

- **Панель инструментов** – сводка по агентам, файлам, инцидентам, графики активности.
- **Машины** – список агентов с онлайн‑статусом, количеством файлов, возможностью массового перезапуска, обновления и полной индексации.
- **Поиск файлов** – полнотекстовый поиск с русской морфологией, фильтрами по агенту, владельцу, расширению, размеру, дате изменения. Контекст найденных совпадений подсвечивается, найденные слова‑триггеры выделяются красным.
- **Инциденты безопасности** – список сработавших триггеров с возможностью отметить их как решённые. Автоматическое обновление каждые 30 секунд.
- **Настройки** – управление списком ключевых слов‑триггеров (добавление, удаление, временное отключение).
- **Админ‑панель** – доступ к стандартной административной панели Django для суперпользователей.

## Управление триггерами

В разделе **«Настройки»** администратор может добавлять, удалять и отключать ключевые слова.  
При обнаружении любого активного слова в содержимом файла создаётся инцидент.  
Поддерживаются фразы из нескольких слов (например, `зеля петух`).

## Полнотекстовый поиск

- Поисковая система использует `SearchVectorField` PostgreSQL с поддержкой русского стемминга (нахождение разных форм одного слова).
- Можно использовать кавычки для точной фразы, операторы `-` для исключения слов.
- Фильтры: выбор одного или нескольких агентов, владелец файла, расширение, диапазон размера, даты изменения.
- Результаты можно экспортировать в CSV.

## Инциденты и уведомления

- Каждый инцидент содержит дату обнаружения, агент, имя файла, сработавшее слово и контекст.
- Статусы: **Новый**, **Подтверждён**, **Решён**.
- Возможна настройка email‑уведомлений и webhook (в планах).

## REST API агента (порт 9090)

Каждый установленный агент предоставляет HTTP‑API для прямого управления и мониторинга:

- `GET /metrics` – метрики агента (количество просканированных файлов)
- `GET /status` – состояние агента в формате JSON (известные файлы, очередь повторных попыток, активные триггеры, конфигурация)
- `POST /scan` – запустить немедленное полное сканирование
- `GET /file?path=<путь>` – скачать оригинальный файл с машины

Эти эндпоинты доступны без авторизации и предназначены для администраторов.

## Архитектура и масштабирование

Серверная часть построена на Docker Compose и включает:
- **веб‑сервер** (Gunicorn + Django) – 2 экземпляра, балансировка через nginx;
- **PostgreSQL 15** с полнотекстовыми индексами;
- **Redis 7** – брокер Celery и кэш сессий;
- **Celery worker + beat** – асинхронная обработка файлов и периодические задачи (отчёты, очистка);
- **nginx** – обратный прокси и раздача статики.

Для более высокой нагрузки можно увеличить количество контейнеров `web` и `celery_worker` через `docker compose up -d --scale web=4 --scale celery_worker=4`.

## Обновление

**Серверная часть:**
```bash
cd afm-server
git pull
sudo bash deploy.sh
```

**Агенты:**
- Массовое обновление через веб‑интерфейс: выберите агентов и нажмите **«Обновить выбранных»**.
- Агент автоматически скачает и установит новый deb‑пакет с сервера, затем перезапустится.
